# AI 行动面板契约

默认页面路径：

- `00 Dashboard/AI 行动面板.md`

该页面是长期存在的固定页面，但内容由 AI 每次按需重算覆盖。

## 页面固定结构

页面按以下顺序生成：

1. 标题
2. 一段简短说明
3. `现在值得推进`
4. `等待条件`
5. `待确认完成`
6. `可忽略 / 暂不处理`
7. `本次更新时向用户确认的判断`

## 标题与说明

标题固定为：

```markdown
# AI 行动面板
```

说明区需要简要写清：

- 本次更新时间
- 本次读取的时间范围
- 这是“行动判断层”，不是正式任务库

## 每条线程的固定字段

每条线程必须包含以下字段：

- `线程`：一句话命名
- `归属`：`项目 / 学生 / 行政 / 未归类主题`
- `状态`：`推进中 / 等待条件 / 待确认完成 / 可忽略`
- `最近证据`：最近一次明确支持该判断的 note / weekly / log
- `建议下一步`：一句或两句可执行动作
- `为什么现在值得关注`：一句说明优先级或保留理由
- `上次判断`：如果有旧面板中的用户判断，则写出；如果没有，可省略

建议格式：

```markdown
### 线程名

- 归属：项目
- 状态：推进中
- 最近证据：2026-W13 周总结；2026-03-30 daily
- 建议下一步：整理训练结果图表，并整合进 Paper 3 正文
- 为什么现在值得关注：已被 recent weekly 标为下周重点，且会影响整条论文收尾节奏
- 上次判断：继续推进
```

## 区块含义

### 现在值得推进

放当前最值得你投入注意力的线程：

- 最近仍在推进
- 有明确下一步
- 或近期是阻塞点 / 关键节点

### 等待条件

放重要但当前推进不了的线程，例如：

- 外部依赖
- 时机未到
- 用户明确说明暂缓
- 需要先完成别的线程

这些事项不能因为两周没推进就直接消失。

### 待确认完成

放从记录看大概率已完成、但信息还不够完全确认的线程。

### 可忽略 / 暂不处理

放当前可降级、不再重要、或已明显失效的线程。

## 更新前询问规则

如果已有旧面板，更新前应先挑少量关键不确定项询问用户。

默认只问：

- 是否已完成
- 是否属于等待条件
- 是否仍然重要

不要逐条问完整张面板。

## 内读源规则

每次更新应写清大致来源范围，例如：

- 上次行动面板之后的新增内容
- 最近 14 天 periodic
- 最近 1 个月 weekly 与关键 logs

用户明确要求时，才扩展到 3 个月。

## 禁止事项

- 不要把页面写成 daily 摘要
- 不要按文件清单罗列来源
- 不要把每条 daily bullet 直接转成线程
- 不要把它写成 TickTick 式待办列表
- 不要默认全量扫描整个 vault
