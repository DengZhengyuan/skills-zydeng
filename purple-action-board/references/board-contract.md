# AI 行动面板契约

默认页面路径：

- `00 Dashboard/AI 行动面板.md`

该页面是长期存在的固定页面，但内容由 AI 在用户确认后按需重算覆盖。

## 页面固定结构

页面按以下顺序生成：

1. 标题
2. 一段简短说明
3. `现在值得推进`
4. `琐碎但重要的小事项`
5. `等待条件`
6. `待确认完成`
7. `可忽略 / 暂不处理`
8. `本次更新时向用户确认的判断`

正式面板不展示候选池，不新增常驻 `候补观察` 区块。

## 标题与说明

标题固定为：

```markdown
# AI 行动面板
```

说明区需要简要写清：

- 本次更新时间
- 本次读取的时间范围
- 这是“行动判断层”，不是正式任务库
- 本次是否使用了“候选审阅模式”

## 每条线程的固定字段

每条线程必须包含以下字段：

- `线程`：一句话命名
- `归属`：`项目 / 学生 / 行政 / 未归类主题`
- `状态`：`推进中 / 等待条件 / 待确认完成 / 可忽略`
- `最近证据`：最近一次明确支持该判断的 note / weekly / log
- `建议下一步`：一句或两句可执行动作
- `为什么现在值得关注`：一句说明优先级或保留理由
- `上次判断`：如果有旧面板中的用户判断，则写出；如果没有，可省略

建议格式：

```markdown
### 线程名

- 归属：项目
- 状态：推进中
- 最近证据：2026-W13 周总结；2026-03-30 daily
- 建议下一步：整理训练结果图表，并整合进 Paper 3 正文
- 为什么现在值得关注：已被 recent weekly 标为下周重点，且会影响整条论文收尾节奏
- 上次判断：继续推进
```

## 小事项区块的固定字段

`琐碎但重要的小事项` 是全页唯一允许不用线程模板的区块。

每条小事项至少应包含以下字段：

- `事项`
- `最近证据`
- `建议动作`
- `为什么暂不升格为线程`
- `上次判断`：如果有旧面板中的用户判断，可写；没有可省略

建议格式：

```markdown
- 事项：联系张志索要 Qing Huang 的论文和旋转流化床图纸
  最近证据：TickTick 遗留任务；2026-03-31 daily
  建议动作：发一轮消息确认资料是否能拿到，并明确缺哪一份
  为什么暂不升格为线程：这是单点推进动作，会支撑后续主线，但本身还不构成长期主题
```

## 区块含义

### 现在值得推进

放当前最值得你投入注意力的线程：

- 最近仍在推进
- 有明确下一步
- 或近期是阻塞点 / 关键节点

### 等待条件

放重要但当前推进不了的线程，例如：

- 外部依赖
- 时机未到
- 用户明确说明暂缓
- 需要先完成别的线程

这些事项不能因为两周没推进就直接消失。

### 琐碎但重要的小事项

放仍有明确动作、值得持续看见，但暂时不足以形成独立主线的事项：

- 有明确下一步
- 对后续工作有帮助
- 但暂时不构成稳定主题，也不会持续牵动多条动作链

如果某项后续反复出现、形成依赖链、或明显牵动主线，应升级为正式线程。

### 待确认完成

放从记录看大概率已完成、但信息还不够完全确认的线程。

### 可忽略 / 暂不处理

放当前可降级、不再重要、或已明显失效的线程。

## 本次更新时向用户确认的判断

这一节必须显式记录本次候选审阅中的关键确认结果。

至少应包括：

- 本次是否使用了候选审阅模式
- 本次确认放掉的旧条目
- 本次确认 `本次不放入` 的新候选
- 本次从旧区块迁移到 `等待条件` 的条目
- 如果有回捞旧记录中的沉底事项，也应说明这是一轮“遗忘事项回捞”判断

这一节是给下次更新读的，不要写成长篇会议纪要。

## 更新前候选审阅规则

如果已有旧面板，正式落板前默认应先做候选审阅，而不是直接覆盖。

正式面板只承接用户已经确认的结果：

- 用户确认保留或纳入的内容，进入对应区块
- 用户确认 `本次不放入` 的新候选，不进入主板，只进入摘要
- 用户确认 `放掉` 的旧条目，不进入主板，只进入摘要

## 内读源规则

每次更新应写清大致来源范围，例如：

- 上次行动面板之后的新增内容
- 最近 14 天 periodic
- 最近 1 个月 weekly 与关键 logs

用户明确要求时，才扩展到 3 个月。

## 禁止事项

- 不要把页面写成 daily 摘要
- 不要按文件清单罗列来源
- 不要把每条 daily bullet 直接转成线程
- 不要把“小事项区”写成全量待办清单
- 不要把它写成待办清单式罗列
- 不要默认全量扫描整个 vault
- 不要把 `候补观察` 写进正式面板
- 不要在未经过用户确认时直接覆盖旧面板
